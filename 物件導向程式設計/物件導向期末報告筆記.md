DevOps 是近年還不斷在興起的一種軟體開發概念

包含了開發者 與 維運人員的之間的一種模式。

DevOps概念中最常使用到的，其中之一就是自動化。自動化包含自動測試，自動包版，自動上版等等，只要是能夠減少人為作業的部分，就能被自動化，使開發者專心在開發上。

DevOps也隸屬於敏捷式開發上的內容之一，所以也必須套用精實開發上的一些原則，像是消除浪費、盡快交付。

Measurement重點就是在透過數據上的反應，來給團隊正向及改進的回饋，例如新功能上線後減少了多少客訴量、縮短多少作業時間等，甚至是每次上線後的BUG率及修復率等，都是可以被測量出來的基本指標，來反應出開發者的量與質是否正比。

除了 Automation 和 Measurement （表示要能實際用數據測量改進 DevOps），Culture ＆Sharing 表示團隊所有成員應該都要有 DevOps 的文化、概念與思維，所有部門與人員之間應該要通透，能有效地溝通，知道大家的目標與各種想法。

+ DevSecOps 的環境、雲端、微服務架構都和傳統的不同，**你真的知道駭客怎麼攻擊？**如果不夠了解現在的

+ DevOps 常用工具和這些環境資安上的威脅，就無法找出真正的資安風險和針對駭客的攻擊方式做預防。

**P11**

方法論

+ 標準化

量化風險

+ 模組化

把資安盡可能也寫成程式和模組化，不管是測試或是資安架構、功能，它也可以讓我們**持續疊代**跟上開發的速度。

+ 自動化

盡可能把自己要做的資安事項和流程自動化



流程

+ 準備階段 : 「資產清點、威脅模型、風險評估、量化」，清點第三方函式庫、雲端服務，接著規劃攻擊者可能的攻擊，定義出潛在問題、並且量化和選擇防禦方法。
+ 設計階段 : 預設有資安系統架構的考量、自有保護機制設計、
+ 建構階段 :  (CI) 靜態掃描
+ 部署階段 :  (CD) 第三方函式庫檢查、動態測試、主機環境與系統配置、機密資訊配置、權限、網路存取
+ 執行階段 : 網路的監控、流量分析、日誌收集與分析。威脅偵測與反應。
